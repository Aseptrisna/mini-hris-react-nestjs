Panduan Lengkap: Membangun Aplikasi SAP dengan Node.js & React

Selamat datang di versi interaktif dari panduan lengkap membangun aplikasi web modern. Bagian ini akan menuntun Anda melalui setiap langkah, mulai dari penyiapan backend hingga pengembangan antarmuka frontend, lengkap dengan cuplikan kode yang dapat disalin.

Stack Teknologi

  • Frontend: React (Vite), Tailwind CSS
  • Backend: Node.js (Express), MongoDB
  • Autentikasi: JSON Web Tokens (JWT)

Backend: Langkah 0 - Prasyarat

Bagian ini akan memandu Anda dalam membangun backend aplikasi menggunakan Node.js dengan framework Express.js. Pastikan Anda sudah menginstal perangkat lunak berikut sebelum melanjutkan.

  • Node.js (versi LTS) dan npm
  • MongoDB (local atau cloud seperti MongoDB Atlas)

Backend: Langkah 1 - Inisialisasi Proyek

Buat folder baru untuk proyek backend Anda dan inisialisasi proyek Node.js menggunakan npm.

mkdir sap-backend-node
cd sap-backend-node
npm init -y

Backend: Langkah 2 - Instalasi Dependensi

Kita akan menginstal Express.js sebagai kerangka server, Mongoose untuk interaksi dengan MongoDB, dan pustaka lain untuk autentikasi dan manajemen environment.

# Dependensi utama
npm install express mongoose jsonwebtoken bcryptjs cors dotenv

# Dependensi development (untuk auto-restart server)
npm install -D nodemon

Backend: Langkah 3 - Struktur Proyek

Membuat struktur folder yang baik akan membuat kode lebih terorganisir dan mudah dikelola.

Backend: Langkah 4 - Konfigurasi Server & Database

Atur file utama `server.js` untuk menjalankan server Express dan menghubungkannya ke MongoDB menggunakan Mongoose.

File: .env

MONGO_URI=mongodb://localhost:27017/sap_db_node
JWT_SECRET=RAHASIA_SUPER_SECRET_JANGAN_DIBAGIKAN
PORT=5000

File: server.js

const express = require('express');
const dotenv = require('dotenv');
const mongoose = require('mongoose');
const cors = require('cors');

dotenv.config();

const app = express();

// Middleware
app.use(cors());
app.use(express.json()); // Untuk parsing application/json

// Koneksi ke Database
mongoose.connect(process.env.MONGO_URI, { useNewUrlParser: true, useUnifiedTopology: true })
  .then(() => console.log('MongoDB Terhubung...'))
  .catch(err => console.error(err));

// Routes (akan kita tambahkan nanti)
app.get('/', (req, res) => {
    res.send('API SAP Berjalan');
});

const PORT = process.env.PORT || 5000;
app.listen(PORT, () => console.log(`Server berjalan di port ${PORT}`));

Tambahkan script ke package.json

Tambahkan skrip `start` dan `dev` untuk memudahkan menjalankan server.

"scripts": {
  "start": "node server.js",
  "dev": "nodemon server.js"
},

Backend: Langkah 5 - Membuat Models (Schema)

Definisikan struktur data menggunakan Mongoose Schema. Ini mirip dengan langkah di NestJS, tetapi implementasinya dalam konteks Express.

File: models/User.js

const mongoose = require('mongoose');
const bcrypt = require('bcryptjs');

const UserSchema = new mongoose.Schema({
    name: { type: String, required: true },
    email: { type: String, required: true, unique: true },
    password: { type: String, required: true },
    role: { 
        type: String, 
        enum: ['SuperAdmin', 'Admin Perusahaan', 'Karyawan'], 
        required: true 
    },
    company: { type: mongoose.Schema.Types.ObjectId, ref: 'Company' }
}, { timestamps: true });

// Hash password sebelum menyimpan
UserSchema.pre('save', async function(next) {
    if (!this.isModified('password')) {
        return next();
    }
    const salt = await bcrypt.genSalt(10);
    this.password = await bcrypt.hash(this.password, salt);
    next();
});

module.exports = mongoose.model('User', UserSchema);

Backend: Langkah 6 - Membuat Routes & Controllers

Definisikan endpoint API di folder `routes` dan pisahkan logikanya di folder `controllers`. Kita juga akan membuat middleware untuk proteksi rute.

Contoh Middleware: middleware/auth.js

const jwt = require('jsonwebtoken');

const protect = (req, res, next) => {
    // ... logika verifikasi token dari header
};

const authorize = (...roles) => {
    return (req, res, next) => {
        if (!roles.includes(req.user.role)) {
            return res.status(403).json({ message: 'Akses ditolak' });
        }
        next();
    };
};

module.exports = { protect, authorize };

Contoh Controller: controllers/companies.js

const Company = require('../models/Company');

// @desc    Buat Perusahaan Baru
// @route   POST /api/v1/companies
// @access  Private/SuperAdmin
exports.createCompany = async (req, res, next) => {
    try {
        const company = await Company.create(req.body);
        res.status(201).json({ success: true, data: company });
    } catch (error) {
        res.status(400).json({ success: false, error: error.message });
    }
};

Contoh Route: routes/companies.js

const express = require('express');
const { createCompany } = require('../controllers/companies');
const { protect, authorize } = require('../middleware/auth');

const router = express.Router();

router.route('/')
    .post(protect, authorize('SuperAdmin'), createCompany);

module.exports = router;

Hubungkan Routes di server.js

// ... di dalam server.js
const companyRoutes = require('./routes/companies');
// ...

// Mount routers
app.use('/api/v1/companies', companyRoutes);
// ...

Frontend: Langkah 5 - Setup API Service & Auth Context

Buat sebuah instance Axios terpusat untuk berkomunikasi dengan backend dan sebuah AuthContext untuk mengelola state autentikasi pengguna (seperti siapa yang login) secara global di seluruh aplikasi.

File: src/context/AuthContext.jsx

import { createContext, useState, useEffect, useContext } from 'react';
import jwt_decode from 'jwt-decode';
import axios from 'axios';

// Buat instance Axios
const apiClient = axios.create({
  baseURL: 'http://localhost:5000/api/v1', // Sesuaikan dengan URL backend Node.js Anda
});

// ... (Sisa kode AuthContext tetap sama) ...

const AuthContext = createContext();

export const AuthProvider = ({ children }) => {
  const [user, setUser] = useState(null);
  const [token, setToken] = useState(() => localStorage.getItem('authToken') || null);

  useEffect(() => {
    if (token) {
      apiClient.defaults.headers.common['Authorization'] = `Bearer ${token}`;
      const decoded = jwt_decode(token);
      setUser({ email: decoded.email, role: decoded.role, id: decoded.id });
    } else {
      delete apiClient.defaults.headers.common['Authorization'];
      setUser(null);
    }
  }, [token]);

  const login = async (email, password) => {
    const response = await apiClient.post('/auth/login', { email, password }); // endpoint mungkin perlu disesuaikan
    const newToken = response.data.token;
    setToken(newToken);
    localStorage.setItem('authToken', newToken);
  };

  const logout = () => { /* ... */ };

  return (
    
      {children}
    
  );
};

export const useAuth = () => {
  return useContext(AuthContext);
};

Cara Menjalankan Proyek

Untuk menjalankan aplikasi secara lokal, Anda perlu menjalankan server backend dan server development frontend secara bersamaan di terminal yang terpisah.

1. Jalankan Backend

# di folder sap-backend-node
npm run dev

2. Jalankan Frontend

# di folder sap-frontend
npm run dev

Pengembangan Lanjutan

Panduan ini adalah fondasi yang kuat. Berikut adalah beberapa ide untuk pengembangan lebih lanjut:

  • Seed Data: Buat script untuk mengisi data awal, seperti akun SuperAdmin.
  • UI/UX: Gunakan library komponen seperti Shadcn/ui atau Headless UI.
  • Manajemen State: Pertimbangkan Zustand atau Redux Toolkit untuk state yang lebih kompleks.
  • Testing: Tulis unit test dan e2e test untuk memastikan kualitas aplikasi.
  • Deployment: Pelajari cara mendeploy backend (Render, Heroku) dan frontend (Vercel, Netlify).
